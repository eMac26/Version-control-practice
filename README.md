# Regarding this Repository 
## What is Version Control?
‘Version Control’ refers to the system which documents changes to files or sets of files gradually. It allows individuals and/or teams to track history, revert to previous versions, and collaborate on source codes. 
## Why Do Developers use Commits? 
### Firstly, what is a ‘Commit’?
A “commit” is a snapshot of your project and its files at a specific point in time. It allows you to save your project in your repository’s history, whilst allowing you to track changes and revert to previous states if needed. This is why developers rely on commits; they need to be able to save their codes, modify it, track changes, and because it consolidates the practice of communicating between other programmers.
